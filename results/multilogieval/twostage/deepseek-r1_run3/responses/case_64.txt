=== REASONING ===
We are given the following:
  Premise: ∀ s, AttendsRegularly s → (SpendsSufficientTimeRevising s → PerformsWellOnExams s)
  Fact1: SpendsSufficientTimeRevising Priya
  Fact2: ¬ PerformsWellOnExams Priya

We want to prove: ¬ AttendsRegularly Priya

Proof idea:
  Assume, for contradiction, that Priya attends regularly (AttendsRegularly Priya).
  Then by the Premise applied to Priya, we have:
      AttendsRegularly Priya → (SpendsSufficientTimeRevising Priya → PerformsWellOnExams Priya)
  So, using our assumption, we get: SpendsSufficientTimeRevising Priya → PerformsWellOnExams Priya.
  But we know Fact1: SpendsSufficientTimeRevising Priya, so we get PerformsWellOnExams Priya.
  However, Fact2 says ¬ PerformsWellOnExams Priya, which is a contradiction.

Therefore, our assumption (AttendsRegularly Priya) must be false, so we have ¬ AttendsRegularly Priya.

In Lean, we can prove this by:
  fun h : AttendsRegularly Priya => 
    let impl := Premise Priya h   -- impl : SpendsSufficientTimeRevising Priya → PerformsWellOnExams Priya
    let performsWell := impl Fact1   -- performsWell : PerformsWellOnExams Priya
    Fact2 performsWell   -- Fact2 performsWell is a proof of False

So the proof term is: fun h => Fact2 (Premise Priya h Fact1)

Alternatively, we can write it as: 
  ¬ AttendsRegularly Priya is equivalent to AttendsRegularly Priya → False
  and we have built a function that takes h (of type AttendsRegularly Priya) and produces False.

Thus, we replace the sorry with: fun h => Fact2 (Premise Priya h Fact1)

Let's write the complete code accordingly.
