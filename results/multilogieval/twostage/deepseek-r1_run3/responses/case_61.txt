- `not_excel_priya`: Priya did not excel in the course.

We want to prove: ¬ AttendsRegularly Priya

How can we prove that Priya did not attend regularly?
Assume by contradiction that Priya attended regularly (AttendsRegularly Priya). Then by the premise applied to Priya, we have:
  StudiesDiligently Priya → ExcelsInCourse Priya
But we know that Priya studied diligently (diligent_priya), so we get ExcelsInCourse Priya.
However, we also have not_excel_priya which says ¬ ExcelsInCourse Priya. This is a contradiction.

Therefore, our assumption (AttendsRegularly Priya) must be false.

In Lean, we can prove this by:
  fun (h_attend : AttendsRegularly Priya) => 
    let h_excel : ExcelsInCourse Priya := premise Priya h_attend diligent_priya
    not_excel_priya h_excel

But note: the theorem is stating `¬ AttendsRegularly Priya`, which is equivalent to `AttendsRegularly Priya → False`.

Alternatively, we can use `by` tactics. However, the problem says we can use either a proof term or `by` tactics. Since the example used a proof term, we can use a term as well.

But note: the example provided in the instructions used a term (R1 Cat T1). So we can write:

Let's break it down:
  We assume `h_attend : AttendsRegularly Priya`.
  Then `premise Priya h_attend` gives us `StudiesDiligently Priya → ExcelsInCourse Priya`.
  Since we have `diligent_priya : StudiesDiligently Priya`, we apply the implication to get `ExcelsInCourse Priya`.
  Then we get a contradiction by `not_excel_priya` applied to that.
