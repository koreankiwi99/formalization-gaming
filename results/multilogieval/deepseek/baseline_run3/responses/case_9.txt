The context provides logical conditions about the party. Let's translate it into Lean.

- We model the party with propositions:
  - `EnoughFood`: There is enough food for everyone.
  - `BringsDish`: At least one guest brings a dish to share.
  - `BirthdayParty`: It is a birthday party (specifically Sam's).
  - `Cake`: There is birthday cake.
  - `IceCream`: There is ice cream.

Axioms:
1. If someone brings a dish, there is enough food: `BringsDish → EnoughFood`.
2. If it's a birthday party, there is cake and ice cream: `BirthdayParty → (Cake ∧ IceCream)`.
3. Either someone brings a dish or there is no cake: `BringsDish ∨ ¬Cake`.

Given: It is Sam's birthday party (`BirthdayParty`).

We need to prove `EnoughFood`.

<lean>
axiom EnoughFood : Prop
axiom BringsDish : Prop
axiom BirthdayParty : Prop
axiom Cake : Prop
axiom IceCream : Prop

axiom A1 : BringsDish → EnoughFood
axiom A2 : BirthdayParty → (Cake ∧ IceCream)
axiom A3 : BringsDish ∨ ¬Cake

axiom T1 : BirthdayParty

theorem sam_has_enough_food : EnoughFood := by
  have cake_ice : Cake ∧ IceCream := A2 T1
  have cake : Cake := cake_ice.left
  cases A3 with
  | inl brings => exact A1 brings
  | inr no_cake => contradiction
</lean>

Proof explanation:
- From `T1` (Sam's birthday) and `A2`, we get `Cake ∧ IceCream`, so `Cake` holds.
- From `A3`, we case split:
  - If `BringsDish` is true, then `A1` gives `EnoughFood`.
  - If `¬Cake` is true, it contradicts `Cake` (from `cake`), so this case is impossible.

Thus, `EnoughFood` is proven.
