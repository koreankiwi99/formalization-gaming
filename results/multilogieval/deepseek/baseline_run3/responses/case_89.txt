- `Monsoon` represents "it is monsoon season".
- `HeavyRains` represents "there are heavy rains".
- `Waters` represents "the farmer waters his crops".
- `GrowStrong` represents "crops grow healthy and strong".
- `Bountiful` represents "the harvest is bountiful".
- `GoodProfit` represents "the farmer makes a good profit".

The axioms are:
1. `A1`: If the farmer waters his crops, they grow strong (`Waters → GrowStrong`).
2. `A2`: If it is monsoon season, there are heavy rains (`Monsoon → HeavyRains`).
3. `A3`: Either the farmer waters his crops daily or there are no heavy rains (`Waters ∨ ¬HeavyRains`).
4. `A4`: If crops grow strong, the harvest is bountiful (`GrowStrong → Bountiful`).
5. `A5`: If the harvest is bountiful, the farmer makes a good profit (`Bountiful → GoodProfit`).

The theorem `target` states that if it is monsoon season, the farmer makes a good profit (`Monsoon → GoodProfit`). The proof uses Lean tactics:
- Assume `Monsoon` (using `intro`).
- Derive `HeavyRains` from `A2`.
- Case analysis on `A3`:
  - If `Waters` is true, derive `GrowStrong` (via `A1`), then `Bountiful` (via `A4`), then `GoodProfit` (via `A5`).
  - If `¬HeavyRains` is true, it contradicts `HeavyRains` (using `contradiction`).

The proof is complete, confirming the theorem.

<lean>
axiom Waters : Prop
axiom GrowStrong : Prop
axiom Monsoon : Prop
axiom HeavyRains : Prop
axiom Bountiful : Prop
axiom GoodProfit : Prop

axiom A1 : Waters → GrowStrong
axiom A2 : Monsoon → HeavyRains
axiom A3 : Waters ∨ ¬ HeavyRains
axiom A4 : GrowStrong → Bountiful
axiom A5 : Bountiful → GoodProfit

theorem target : Monsoon → GoodProfit := by
  intro hMonsoon
  have hRains : HeavyRains := A2 hMonsoon
  cases A3
  case inl hWaters =>
    have hStrong : GrowStrong := A1 hWaters
    have hBountiful : Bountiful := A4 hStrong
    exact A5 hBountiful
  case inr noRains =>
    contradiction
</lean>
