The compilation error occurred because the `contradiction` tactic didn't automatically resolve the conflict between `great_service` and `¬great_service` in the `inl` branch. The solution is to explicitly construct the contradiction using `False.elim` with the conflicting hypotheses.

Here's the corrected Lean 4 code:

<lean>
axiom great_service : Prop
axiom reputation_grows : Prop
axiom host_events : Prop
axiom more_customers : Prop
axiom revenue_improves : Prop

axiom Premise1 : great_service → reputation_grows
axiom Premise2 : host_events → more_customers
axiom Premise3 : great_service ∨ host_events
axiom Premise4 : more_customers → revenue_improves

axiom service_decreased : ¬great_service

theorem revenue_result : revenue_improves := by
  have host_true : host_events := by
    cases Premise3 with
    | inl great => exact False.elim (service_decreased great)
    | inr host => exact host
  have more_cust : more_customers := Premise2 host_true
  exact Premise4 more_cust
</lean>

The proof demonstrates:
1. From `service_decreased` (¬great_service) and the disjunction `Premise3`, we deduce `host_events` must be true
2. Using `Premise2`, `host_events` implies `more_customers`
3. Using `Premise4`, `more_customers` implies `revenue_improves`

Thus, despite decreased service quality, revenue improved due to hosting events.
