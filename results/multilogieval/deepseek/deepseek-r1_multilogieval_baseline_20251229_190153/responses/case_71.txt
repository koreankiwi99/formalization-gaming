We'll model the given context and question in Lean. The key entities are the weather condition (sunny), the state of the pool water (warm), and Alex's actions and states (applying sunscreen, getting burnt, going swimming, getting exercise). The rules from the context are:

1. If someone applies sunscreen, they don't get burnt (∀ person).
2. If someone goes swimming, they get exercise (∀ person).
3. For Alex: either applied sunscreen or did not get exercise.
4. If pool water is warm, then everyone goes swimming.
5. If weather sunny, then pool water is warm.

Additionally, we know:
- Alex got sunburnt (given in the question).
- The context states "It was a sunny day", but careful analysis shows that when Alex gets sunburnt, we derive a contradiction if sunny is true. Therefore, for logical consistency, we must not include "sunny" as an axiom, and instead deduce its truth value from the other facts.

Steps of reasoning in Lean:
- We have `get_burnt Alex` (Alex got sunburnt).
- By Rule 1, Alex did not apply sunscreen (since sunburnt).
- By Rule 3 (sunscreen or no exercise) and not applying sunscreen, Alex had no exercise.
- By Rule 2 (swimming implies exercise) and no exercise, Alex did not go swimming.
- By Rule 4 (if pool warm, everyone swims) and Alex not swimming, pool not warm.
- By Rule 5 (sunny implies pool warm) and pool not warm, weather not sunny.

Thus, from Alex sunburnt, we deduce that the weather was not sunny. In Lean, this is provable.

<lean>
axiom Person : Type
axiom Alex : Person

axiom sunny : Prop
axiom apply_sunscreen : Person → Prop
axiom get_burnt : Person → Prop
axiom go_swimming : Person → Prop
axiom get_exercise : Person → Prop
axiom pool_water_warm : Prop

-- Rule 1: Applying sunscreen prevents sunburn
axiom sunscreen_prevents_burn : ∀ p, apply_sunscreen p → ¬ get_burnt p

-- Rule 2: Swimming causes exercise
axiom swimming_causes_exercise : ∀ p, go_swimming p → get_exercise p

-- Rule 3: For each person, sunscreen or no exercise (but for Alex, used in proof)
axiom sunscreen_or_no_exercise : ∀ p, apply_sunscreen p ∨ ¬ get_exercise p

-- Rule 4: Warm pool water implies everyone swims
axiom warm_pool_implies_swimming : pool_water_warm → ∀ p, go_swimming p

-- Rule 5: Sunny weather implies warm pool
axiom sunny_implies_warm_pool : sunny → pool_water_warm

-- Given: Alex got sunburnt
axiom alex_burnt : get_burnt Alex

-- Prove weather was NOT sunny (using the rules and Alex being burnt)
theorem not_sunny : ¬ sunny :=
  fun h_sunny : sunny =>
  have warm_pool : pool_water_warm := sunny_implies_warm_pool h_sunny
  have everyone_swims : ∀ p, go_swimming p := warm_pool_implies_swimming warm_pool
  have alex_swims : go_swimming Alex := everyone_swims Alex
  have alex_exercise : get_exercise Alex := swimming_causes_exercise Alex alex_swims
  have alex_sunscreen_or_not : apply_sunscreen Alex ∨ ¬ get_exercise Alex := sunscreen_or_no_exercise Alex
  Or.elim alex_sunscreen_or_not
    -- Case 1: Alex applied sunscreen -> no burn (contradiction to alex_burnt)
    (fun sunscreen : apply_sunscreen Alex =>
      have no_burn : ¬ get_burnt Alex := sunscreen_prevents_burn Alex sunscreen
      no_burn alex_burnt)
    -- Case 2: Alex had no exercise -> false since we have alex_exercise
    (fun no_exercise : ¬ get_exercise Alex => no_exercise alex_exercise)
</lean>
