The reasoning is formalized by defining the necessary types, objects, and axioms based on the context. The key is to model the logical dependencies, such as the implication between the stove being on and the kitchen getting hot, the relationship between pancake making and stove usage, the disjunction about the stove state, and how hunger and dinner affect pancake making.

- `KitchenHot` and `StoveOn` are propositions representing the kitchen's state and the stove's state, respectively.
- `Person` is a type representing people, with `Sam` as a constant instance.
- Predicates like `hungry`, `makes_pancakes`, `uses_stove`, and `ate_dinner` model the conditions per person.
- Axioms capture the premises:
  - `A1`: Stove on implies kitchen hot.
  - `A3`: Disjunction: stove is on or no one used the stove.
  - `A4`: If a person is hungry, they make pancakes.
  - `A5`: If Sam didn't eat dinner, he was hungry.
  - `A6`: If a person makes pancakes, they use the stove.

The theorem `final` proves that if the kitchen did not get hot, then Sam ate dinner. The proof:
1. If the kitchen is not hot (`h_not_hot`), the stove must be off (`h_noStove`) by contrapositive of `A1`.
2. With the stove off (`h_noStove`), and given `A3`, no one used the stove (`h_nouse`).
3. This means no person uses the stove (`h_nouse_all`).
4. By contradiction: assume Sam didn't eat dinner (`h_notdinner`). 
5. Then Sam is hungry (`h_hungry` by `A5`).
6. Then Sam makes pancakes (`h_makes` by `A4`).
7. Then Sam uses the stove (`h_uses` by `A6`).
8. This contradicts `h_nouse_all`, which states no person uses the stove.

Thus, the original assumption (Sam didn't eat dinner) is false. Lean 4's classical logic resolves the double negation.

<lean>
axiom Person : Type
axiom Sam : Person

axiom KitchenHot : Prop
axiom StoveOn : Prop

axiom hungry : Person → Prop
axiom makes_pancakes : Person → Prop
axiom uses_stove : Person → Prop
axiom ate_dinner : Person → Prop

axiom A1 : StoveOn → KitchenHot
axiom A3 : StoveOn ∨ ¬ (∃ p : Person, uses_stove p)
axiom A4 : ∀ p : Person, hungry p → makes_pancakes p
axiom A5 : ¬ ate_dinner Sam → hungry Sam
axiom A6 : ∀ p : Person, makes_pancakes p → uses_stove p

theorem final : ¬ KitchenHot → ate_dinner Sam := by
  intro h_not_hot
  have h_noStove : ¬ StoveOn := fun h_on => h_not_hot (A1 h_on)
  have h_nouse : ¬ ∃ p : Person, uses_stove p :=
    match A3 with
    | Or.inl h_on => False.elim (h_noStove h_on)
    | Or.inr h => h
  have h_nouse_all : ∀ p : Person, ¬ uses_stove p := 
    fun p => mt (fun use_p => ⟨p, use_p⟩) h_nouse
  by_contra h_notdinner
  have h_hungry : hungry Sam := A5 h_notdinner
  have h_makes : makes_pancakes Sam := A4 Sam h_hungry
  have h_uses : uses_stove Sam := A6 Sam h_makes
  exact h_nouse_all Sam h_uses
</lean>
