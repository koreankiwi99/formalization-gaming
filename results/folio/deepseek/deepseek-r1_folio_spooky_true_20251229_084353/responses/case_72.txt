The reasoning follows these steps:
1. KO is a mature stock (premise), so it's suitable for retirement (from "All mature stocks are suitable").
2. Growth stocks are bought for appreciation (from "All growth stocks are bought...").
3. Stocks bought for appreciation aren't suitable for retirement (premise).
4. Therefore, if KO were a growth stock:
   - It would be bought for appreciation
   - Then not suitable for retirement
   - But it is suitable (step 1) → contradiction
5. Similarly, if KO were bought for appreciation (even if not growth stock):
   - Then not suitable for retirement
   - But it is suitable → contradiction
6. So the antecedent (KO is growth/bought for appreciation) must be false
7. Thus the implication holds (false antecedent makes implication true)

Lean proof implements this by:
- Using contradiction in both cases of disjunction
- Resulting false proves anything (including conclusion)

<lean>
axiom obj : Type
axiom KO : obj
axiom GrowthStock : obj → Prop
axiom BoughtForAppreciation : obj → Prop
axiom SuitableForRetirement : obj → Prop
axiom MatureStock : obj → Prop
axiom Stock : obj → Prop
axiom PriceVolatile : obj → Prop
axiom A1 : ∀ x, GrowthStock x → BoughtForAppreciation x
axiom A2 : ∀ x, BoughtForAppreciation x → ¬ SuitableForRetirement x
axiom A3 : ∃ x, Stock x ∧ GrowthStock x
axiom A4 : ∀ x, MatureStock x → SuitableForRetirement x
axiom A5 : MatureStock KO
axiom A6 : ∀ x, MatureStock x → Stock x
theorem ko_statement : (GrowthStock KO ∨ BoughtForAppreciation KO) → (¬ Stock KO ∧ ¬ PriceVolatile KO) := 
  fun h => 
    have suitable_KO : SuitableForRetirement KO := A4 KO A5
    have stock_KO : Stock KO := A6 KO A5
    Or.elim h 
      (fun growthKo : GrowthStock KO =>
        have : BoughtForAppreciation KO := A1 KO growthKo
        have : ¬ SuitableForRetirement KO := A2 KO this
        absurd suitable_KO this
      )
      (fun boughtKo : BoughtForAppreciation KO =>
        have : ¬ SuitableForRetirement KO := A2 KO boughtKo
        absurd suitable_KO this
      )
</lean>
